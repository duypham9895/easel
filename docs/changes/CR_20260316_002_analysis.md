# CR_20260316_002 — Calendar Screen Rebuild: Cycle Timeline

## Change ID
`CR_20260316_002`

## Title
Rebuild Calendar screen as a focused Cycle Timeline with tap-to-log interaction

## Requestor
Product (Edward) — 2026-03-16

## Current Behavior

### Calendar screen currently contains:
1. **Calendar grid** — `react-native-calendars` with custom `CalendarDayCell`, color-coded markers (period logged/predicted, fertile, ovulation), range bands, flow dots. Tap any day → opens inline `PeriodLogPanel`.
2. **Legend** — 5 items (Period logged, Period predicted, Fertile window, Ovulation day, Today)
3. **PeriodLogPanel (inline)** — appears below calendar when a day is tapped. Contains: flow intensity selector, symptom chips, factors/tags, notes, save, delete.
4. **MyCycleCard** — cycle stats (avg cycle/period length, next period), period history list, **"Log a period" button → redirects to `/health-sync`**, "Edit cycle settings" link → redirects to `/health-sync`
5. **PredictionWindowCard** — shows predicted next period window with confidence level
6. **Tap hint** — "Tap any day to log your period" for users with no logs
7. **Sun's PartnerCalendarView** — read-only calendar with predictions for linked partner

### Problems identified:
- **Problem 1 — "Log how she is feeling today"**: NOT present on Calendar screen. DailyCheckIn is only on MoonDashboard. Already resolved. No action needed.
- **Problem 2 — "Log a period" button mislabeling**: MyCycleCard's "Log a period" button (`onLogPeriod`) redirects to `/health-sync` instead of actually logging a period. Confusing and misleading.
- **Problem 3 — No clear purpose**: MyCycleCard clutters the Calendar with cycle stats and period history that belong in Settings or a dedicated history view. The Calendar should focus on visual timeline + direct day-level logging.

## Desired Behavior

### Calendar screen becomes "Cycle Timeline":
1. **Calendar grid** — KEEP as-is (CalendarDayCell with color coding)
2. **Legend** — KEEP as-is
3. **Tap behavior change**:
   - Tap unlogged past/today day → auto-mark as period (default flow = medium, via `savePeriodDayLog`)
   - Tap already-logged day → open bottom sheet (PeriodLogPanel converted to Modal bottom sheet)
   - Future days → non-tappable
4. **MyCycleCard** — REMOVE entirely
5. **PredictionWindowCard** — KEEP
6. **Tap hint** — KEEP (existing `tapToLog` / `firstTimeHint`)
7. **Bottom sheet for editing** — convert PeriodLogPanel to Modal bottom sheet (reuse PeriodLogSheet's Modal pattern)

## Delta Statement
Change the Calendar screen from a mixed-purpose view (calendar + cycle stats + broken redirect) to a focused cycle timeline by removing MyCycleCard and converting the tap-to-log interaction to: first-tap-marks, second-tap-opens-sheet.

## Business Justification
The current "Log a period" button misleads users by redirecting to health-sync instead of logging. MyCycleCard duplicates information available elsewhere. Simplifying to a single-purpose timeline with direct tap-to-log improves trust and usability.
