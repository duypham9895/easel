## Test Execution Notes

1. **Device-test cases** require physical iPhone for haptic verification. Simulator acceptable for visual/layout checks only.
2. **Unit-test cases** (CV-16, BC-02, RG-02) run via `npm test` in the `app/` directory. Must pass in CI before merge.
3. **Code-inspection cases** (CV-16) verify database state via Supabase dashboard or SQL query on staging.
4. **Partner sync tests** (CV-13, EC-07) require two devices simultaneously logged in as Moon and Sun respectively.
5. **Offline tests** (EC-06) require toggling airplane mode on a physical device with Supabase staging environment.
6. **i18n tests** (EC-10) require switching device language to Vietnamese in iOS Settings before launching the app.
7. All P0 cases must pass on both iPhone 15 Pro and iPhone SE before release signoff.
