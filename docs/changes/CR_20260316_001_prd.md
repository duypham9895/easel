## 1. Problem Statement

Easel's current period logging experience treats cycle tracking as data entry rather than a health companion interaction. Specific pain points:

1. **Binary start/end toggle.** Moon can only mark "period started" or "period ended" on a single calendar day. There is no way to select a range of days visually, no way to go back and annotate individual days within a period, and no way to record how the period changed day-to-day.

2. **No flow intensity tracking.** The `period_logs` table stores `start_date`, `end_date`, `tags[]`, and `notes` — but nothing about per-day flow (light, medium, heavy, spotting). This is the single most-requested data point in every major competitor (Flo, Clue, Natural Cycles). Without it, Easel cannot meaningfully improve predictions or give Moon a sense that the app "understands" her body.

3. **No per-day symptom capture during period.** The existing `daily_logs` table supports mood, energy, and freeform symptoms, but it is disconnected from the period flow. Moon must navigate to a separate check-in to record cramps, headache, or bloating — there is no contextual prompt while she is already in the period logging flow.

4. **Confirmation via `Alert.alert()`.** Every save and delete action triggers a native alert dialog. This breaks immersion, feels clinical, and provides zero visual delight. Competitors use inline confirmation with animated checkmarks and haptic feedback.

5. **No visual feedback loop.** After logging, nothing changes visually on the calendar except a dot color. There is no summary card, no "your period lasted X days" insight, no celebration micro-animation. Moon has no reason to come back and update her log.

6. **Sun receives no real-time detail.** Sun gets a Realtime event when Moon inserts or updates a `period_log`, but the payload contains only `start_date` and `end_date`. There is no notification that Moon just logged flow intensity or symptoms — so Sun's dashboard cannot adapt to "she's having a heavy day" versus "spotting, almost done."

The net effect: Easel's core value proposition (empathetic couples tracking) is undermined by a logging experience that feels like a spreadsheet rather than a health companion.

## 4. User Stories

### US-1: Tap-to-log calendar days
**As Moon**, I want to tap individual calendar days to mark them as period days, so I can visually build my period range without navigating to a separate form.

**Acceptance criteria:**
- Tapping a non-period day within the current or previous cycle marks it as a period day with a default flow of "medium."
- Tapping an already-marked period day deselects it (removes that day from the period).
- Multi-day selection is contiguous — the system auto-fills gaps if Moon taps day 1 and day 5 (with a confirmation prompt).
- Visual feedback: selected days animate with a ripple effect and the menstrual phase color fills the cell.
- Validation: cannot select future dates, cannot select dates > 30 days in the past, cannot overlap with another logged period.

### US-2: Record flow intensity per day
**As Moon**, I want to record how heavy my flow is each day (spotting, light, medium, heavy), so I can track patterns and share context with my partner.

**Acceptance criteria:**
- After tapping a period day, a bottom sheet slides up showing four flow options with distinct icons and colors.
- Default selection is "medium" when first marking a day.
- Moon can change flow intensity on any previously logged day by tapping it again.
- Flow data is stored per-day (not per-period) in a new `period_day_details` table.
- Calendar cells show a visual indicator of flow level (dot size or shade intensity).

### US-3: Track symptoms during period
**As Moon**, I want to quickly tag symptoms (cramps, headache, bloating, fatigue, back pain, mood swings) while logging my period, so I don't have to navigate to a separate daily check-in.

**Acceptance criteria:**
- The period day detail sheet includes a horizontally scrollable symptom picker with 8-12 predefined symptoms.
- Moon can select multiple symptoms per day.
- Symptom data is stored in `period_day_details.symptoms[]`.
- Symptoms display as small icons/pills beneath the flow indicator on the calendar.
- The existing `daily_logs` symptom field continues to work independently (this is additive, not a replacement).

### US-4: Partner real-time sync for period details
**As Sun**, I want to see when Moon has logged her period — including flow intensity — in real-time, so I can understand how she is feeling today and respond with appropriate support.

**Acceptance criteria:**
- Sun's dashboard updates within 2 seconds of Moon saving period details (via Supabase Realtime).
- Sun sees a contextual card: "Moon is on day 3 — heavy flow" instead of just "Moon is on her period."
- Sun's phase-aware guidance adapts: heavy flow days surface different advice than spotting days.
- Sun cannot edit Moon's period data (read-only via RLS).
- Flow and symptom detail is visible on Sun's calendar view (PartnerCalendarView).

### US-5: Visual confirmation and delight on save
**As Moon**, I want clear, beautiful visual feedback when I save my period log, so I feel confident my data was recorded and motivated to keep tracking.

**Acceptance criteria:**
- Saving triggers an inline animated checkmark (Lottie or Reanimated) — no `Alert.alert()`.
- A haptic success pulse fires on save (existing `notificationSuccess()`).
- A brief summary toast appears: "Period logged: Mar 10-14, 5 days, mostly heavy flow."
- The calendar smoothly re-renders with updated markers (no flicker, no full re-mount).
- Deleting a period log shows a destructive confirmation bottom sheet (not `Alert.alert()`), with an undo option for 5 seconds.

## 7. Technical Design Overview

### 7.1 Database Migration (008)

```sql
-- 008_period_day_details.sql
CREATE TABLE public.period_day_details (
  id              UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  user_id         UUID NOT NULL REFERENCES public.profiles(id),
  period_log_id   UUID NOT NULL REFERENCES public.period_logs(id) ON DELETE CASCADE,
  log_date        DATE NOT NULL,
  flow_intensity  TEXT NOT NULL DEFAULT 'medium'
                    CHECK (flow_intensity IN ('spotting', 'light', 'medium', 'heavy')),
  symptoms        TEXT[],
  notes           TEXT,
  created_at      TIMESTAMPTZ NOT NULL DEFAULT NOW(),
  updated_at      TIMESTAMPTZ NOT NULL DEFAULT NOW(),

  UNIQUE (user_id, log_date)
);

ALTER TABLE public.period_day_details ENABLE ROW LEVEL SECURITY;

CREATE POLICY "period_day_details: read own or partner's"
  ON public.period_day_details FOR SELECT
  USING (user_id = auth.uid() OR user_id = public.my_partner_id());

CREATE POLICY "period_day_details: write own only"
  ON public.period_day_details FOR ALL
  USING (user_id = auth.uid())
  WITH CHECK (user_id = auth.uid());

CREATE TRIGGER period_day_details_updated_at
  BEFORE UPDATE ON public.period_day_details
  FOR EACH ROW EXECUTE FUNCTION public.set_updated_at();

ALTER PUBLICATION supabase_realtime ADD TABLE public.period_day_details;
```
