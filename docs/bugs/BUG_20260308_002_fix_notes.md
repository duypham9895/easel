# BUG_20260308_002 — Fix Notes

## Fix Summary

Removed the conditional `if` block from the `deploy-functions` job in `.github/workflows/supabase.yml` so that Supabase Edge Functions deploy on every push to `main`, matching the existing `migrate` job behavior.

## Change Details

**File:** `.github/workflows/supabase.yml`

**Before (lines 54-60):**
```yaml
deploy-functions:
  name: Edge Functions
  runs-on: ubuntu-latest
  if: |
    github.event_name == 'workflow_dispatch' ||
    contains(toJson(github.event.commits.*.modified), 'app/supabase/functions/') ||
    contains(toJson(github.event.commits.*.added), 'app/supabase/functions/')
```

**After:**
```yaml
deploy-functions:
  name: Edge Functions
  runs-on: ubuntu-latest
  # No condition — runs on every push (idempotent, same as migrate job)
```

## Why This Fix Works

1. `supabase functions deploy` is **idempotent** — it checksums function source and only re-deploys if code changed. The same reasoning already applies to the `migrate` job (`supabase db push` is idempotent).
2. Removes the fragile `contains(toJson(github.event.commits.*.modified), ...)` pattern that fails for large multi-file commits due to GitHub payload truncation.
3. Ensures edge functions are always deployed, preventing the silent 404 failures that have been occurring daily.
4. Overhead is minimal (~30s per CI run for 2 functions).
