# BUG_20260308_001 — Display Name TextInput placeholder truncated and letter-spaced

## Environment

| Field | Value |
|-------|-------|
| Platform | iOS 26.3, iPhone 17 Pro |
| App Version | v1.5.1 |
| Expo SDK | 52 |
| React Native | 0.76 |
| Screen | Settings (`app/app/(tabs)/settings.tsx`) |
| Affected Line | 157 |

## Steps to Reproduce

1. Open the app as either Moon or Sun role.
2. Navigate to the **Settings** tab.
3. Ensure the **Display Name** field is empty (no saved name).
4. Observe the placeholder text in the TextInput.

## Expected Behavior

The placeholder text **"Your name"** displays fully and legibly inside the TextInput, with normal letter-spacing, matching the visual density and font size of adjacent rows (Email, Language, Role).

## Actual Behavior

The placeholder text renders as **"Y o u r  n a"** — characters are visually spread apart with abnormal letter-spacing, and the text is truncated (the full string "Your name" does not fit). The check button (Feather `check` icon) adjacent to the input is either barely visible or pushed off-screen.

## Severity Classification

| Attribute | Assessment |
|-----------|------------|
| **Severity** | **Medium (P2)** |
| **Impact** | Visual/UX — the field appears broken and confusing. Users may not understand what to type or may think the app is buggy. Does not block functionality (typing still works). |
| **Frequency** | 100% reproducible — every user sees this on first visit to Settings before entering a display name. |
| **Affected Users** | All users (both Moon and Sun roles). Every new user who has not yet set a display name will see the broken placeholder. |

## Root Cause Analysis

The TextInput at line 157 applies an inline style override `{ width: 120, padding: 6 }` on top of `settingStyles.input`:

```tsx
style={[settingStyles.input, { width: 120, padding: 6 }]}
```

**Computed style breakdown:**

| Property | Source | Value |
|----------|--------|-------|
| `fontSize` | `Typography.body` via `settingStyles.input` | 16 |
| `width` | Inline override | 120 |
| `padding` | Inline override (overrides `Spacing.sm = 8`) | 6 |
| `backgroundColor` | `settingStyles.input` | `Colors.inputBg` |
| `borderRadius` | `settingStyles.input` | `Radii.sm` (12) |

With `padding: 6` on both sides, the usable content width is `120 - 12 = 108px`. At `fontSize: 16`, the string "Your name" requires approximately 130-140px to render. React Native's TextInput attempts to fit the placeholder into the undersized container, resulting in character truncation and visual letter-spacing artifacts.

**Contrast with working rows:** The Email, Language, and Role rows all use the `Row` component which renders values with `rowStyles.value` (`Typography.bodyBold`) inside `rowStyles.row` (`flexDirection: 'row', justifyContent: 'space-between'`). Critically, these rows do **not** apply a fixed `width` — the text is allowed to flex naturally within the row, so it never truncates.

**The fix** should remove the hardcoded `width: 120` and instead use `flex: 1` or `minWidth` to allow the TextInput to size naturally within its parent flex row. The parent container (line 155) already uses `flexDirection: 'row'` with `gap: 8`, so the input can flex to fill available space while leaving room for the check/save button.

## Related Features

- **Display Name editing** — the TextInput is used to edit the user's display name, which appears in the partner's dashboard.
- **Settings screen layout** — other TextInputs in the same screen (cycle length, period length at lines 534-540) use `settingStyles.input` without `width: 120` and render correctly.
- **i18n** — the placeholder comes from `t('yourName')`. In Vietnamese, the equivalent string may be shorter or longer, making a fixed width even more problematic across locales.
