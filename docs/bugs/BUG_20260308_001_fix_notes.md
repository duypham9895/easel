# BUG_20260308_001 — Fix Notes

## Fix Summary

Replaced hardcoded `width: 120` with `flex: 1` on the Display Name TextInput, allowing it to expand naturally within its flex row. Added `textAlign: 'right'` for visual consistency with other value fields.

## Change Details

**File:** `app/app/(tabs)/settings.tsx`, line 157

**Before:**
```tsx
style={[settingStyles.input, { width: 120, padding: 6 }]}
```

**After:**
```tsx
style={[settingStyles.input, { flex: 1, padding: 6, textAlign: 'right' }]}
```

## Why This Fix Works

1. `flex: 1` lets the TextInput expand to fill available space in the parent flex row (which has `flexDirection: 'row', alignItems: 'center', gap: 8`)
2. The check button icon (18px + gap) retains its intrinsic size as a flex sibling
3. `textAlign: 'right'` aligns input text/placeholder to the right, matching adjacent rows (Email, Language, Role)
4. Works for all locales — no magic numbers to break with longer translations
5. Works for all screen sizes — flexbox handles distribution
